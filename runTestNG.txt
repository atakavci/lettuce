Running TestNG Tests in Maven
1. Add Dependencies
xml<dependency>
    <groupId>org.testng</groupId>
    <artifactId>testng</artifactId>
    <version>7.8.0</version>
    <scope>test</scope>
</dependency>
2. Maven Surefire Plugin (runs tests automatically)
xml<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-surefire-plugin</artifactId>
            <version>3.1.2</version>
        </plugin>
    </plugins>
</build>
Surefire auto-detects TestNG if it's on the classpath — no extra config needed for basic usage.

3. Run Commands
bash# Run all tests
mvn test

# Run a specific test class
mvn test -Dtest=MyTestClass

# Run a specific test method
mvn test -Dtest=MyTestClass#myMethod

# Skip tests
mvn install -DskipTests

4. Using a testng.xml Suite File (optional but common)
xml<!-- testng.xml -->
<!DOCTYPE suite SYSTEM "https://testng.org/testng-1.0.dtd">
<suite name="MySuite">
    <test name="MyTests">
        <classes>
            <class name="com.example.MyTestClass"/>
        </classes>
    </test>
</suite>
Point Surefire to it in pom.xml:
xml<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-surefire-plugin</artifactId>
    <version>3.1.2</version>
    <configuration>
        <suiteXmlFiles>
            <suiteXmlFile>src/test/resources/testng.xml</suiteXmlFile>
        </suiteXmlFiles>
    </configuration>
</plugin>

5. Run Specific Groups
xml<!-- Tag your tests -->
@Test(groups = {"smoke", "regression"})
public void myTest() { ... }
xml<!-- pom.xml - only run smoke group -->
<configuration>
    <groups>smoke</groups>
</configuration>
Or from command line:
bashmvn test -Dgroups=smoke
```

```
mvn test
   └── Surefire Plugin
         └── Detects TestNG on classpath
               └── Runs tests → target/surefire-reports/
Reports are generated in target/surefire-reports/ as XML and HTML after each run.
